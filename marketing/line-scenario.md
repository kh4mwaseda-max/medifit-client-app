# AYF LINE ステップ配信シナリオ
**担当: Kira（リサーチ） + 橘凜（設計）**
**作成日: 2026-04-12**
**根拠: research-3-line-marketing.md**

---

## 基本方針

- 登録直後〜7日間で「使う習慣」を定着させる
- 通知は1日1回まで。朝or夜のどちらかに絞る
- AYFの価値（「なぜ？」が分かる）を体感させることが目的
- ブロック率を下げるため、送らなくていい日は送らない

---

## シナリオ設計

### Day 0（登録直後・自動）

```
🎉 AllYourFitへようこそ！

まず最初にやること：
① あすけん・STRONG・筋トレMemoなど、いつものアプリのスクショをここに送ってください
② AIが自動で読み取ってダッシュボードに記録します

試しに1枚送ってみてください👇
```

**目的**: 最初のアクションを即座に促す。登録→離脱を防ぐ。

---

### Day 1（翌日朝 8:00）

```
おはようございます！

昨日のデータは記録できましたか？

💡 AYFのポイント：
スクショを送り続けると「体重が動かない理由」が自動でグラフになります。
最低3日分のデータがたまると、相関分析が始まります。

今日も1枚送ってみてください📸
```

**目的**: 継続のハードルを下げる。「3日」という短い目標を設定。

---

### Day 3（3日目夜 21:00）

```
3日間、記録できていますか？

3日分のデータがたまったら、ダッシュボードで「体重 × カロリー収支」のグラフを確認してみてください。

👉 [ダッシュボードを開く]（リンク）

「カロリーは足りているのに体重が落ちない」
「タンパク質が少なかった日は翌日しんどかった」

こういうパターンが見え始めます。
```

**目的**: ダッシュボードへの誘導。初回の「なるほど体験」を作る。

---

### Day 5（5日目朝 8:00）

送信条件: **過去48時間にスクショ送信がない場合のみ**

```
最近スクショ送れてますか？

記録が途切れると分析精度が落ちてしまいます。

忙しい日は1枚でもOKです。
体重だけ・食事だけでも記録できます📊
```

**目的**: 離脱しかけたユーザーへの再エンゲージ。プレッシャーをかけすぎない。

---

### Day 7（7日目夜 20:00）

```
1週間、お疲れ様でした！

7日分のデータがたまったユーザーには、今週の自動レポートをお届けします。

📊 今週のサマリー
• 記録日数: （自動挿入）
• 平均カロリー: （自動挿入）
• 体重変化: （自動挿入）

来週も一緒に頑張りましょう💪
```

**目的**: 1週間の達成感を作る。「継続している」実感を強化。

---

### 以降（週次・任意）

毎週月曜朝に前週のサマリーレポートを送信。
ただし「記録なし週」は送信スキップ（通知疲れを防ぐ）。

## トレーナープラン用シナリオ（別フロー）

### トレーナー登録直後

```
トレーナー登録が完了しました✅

まずやること：
① 「招待リンク発行」からクライアントを招待
② クライアントにLINEで招待リンクを送る
③ クライアントが登録したら通知が届きます

クライアントが登録すると、スクショ送信→自動解析がスタートします。
```

### クライアント登録通知（リアルタイム）

```
📋 [クライアント名] さんが登録しました！

ダッシュボードで状況を確認できます：
👉 [ダッシュボードへ]
```

## 実装メモ（佐藤へ）

- Day 0・クライアント登録通知: `pushMessage()` で即時送信（実装済み）
- Day 1, 3, 5, 7: スケジュール配信が必要 → Supabase Edge Functions + pg_cron で実装可能
- Day 5 の条件付き送信: `line_messages` テーブルで最終送信日時を管理して判定
- 週次レポート: 既存の assessment 生成ロジックと連携
