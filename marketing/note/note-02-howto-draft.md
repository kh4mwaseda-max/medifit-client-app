## 作ったもの

**Client Fit（Client Fit）**

クライアントがLINEにスクショを送るだけで、食事×トレーニング×体重を統合分析してくれるダッシュボード。

従来のトレーナーの管理フロー：
```
クライアントが個人LINEに写真を送る
→ トレーナーが目視でチェック
→ Excelやメモに手打ち
→ 毎週手動でレポート作成
```

Client Fitのフロー：
```
クライアントがClient FitのLINEにスクショを送る
→ AIが自動解析
→ ダッシュボードに反映
→ 週次レポート自動生成
```

## 作り方：全体の流れ

```
Step 1: 環境を整える（30分）
Step 2: Claude Codeに設計を相談する（1時間）
Step 3: データベースを作る（1〜2時間）
Step 4: LINEボットを繋げる（1〜2時間）
Step 5: AIのスクショ解析を実装する（2〜3時間）
Step 6: ダッシュボードを作る（2〜4時間）

```

全部で、集中してやれば2〜3日。週末2回あれば完成する。

---

## Step 1: 環境を整える

まず以下をインストールする。

**Node.js**
`https://nodejs.org` からインストール。LTS版でOK。

**Claude Code**
ターミナル（Macならターミナル.app、WindowsならPowerShell）で：
```
npm install -g @anthropic-ai/claude-code
```
インストール後、Anthropicのアカウントでログインする。

**Supabaseのアカウント作成**
`https://supabase.com` でアカウント作成。新しいプロジェクトを作成する。

**Vercelのアカウント作成**
`https://vercel.com` でアカウント作成（GitHubアカウントで登録するのが楽）。

---

## Step 2: Claude Codeに設計を相談する

ここが一番のポイントだ。

Claude Codeを起動して、こう伝える：

```
パーソナルトレーナー向けのクライアント管理アプリを作りたい。

要件：
- クライアントがLINEにスクショを送ると、AIが食事や筋トレ内容を解析する
- トレーナーがダッシュボードで全クライアントの状況を一覧できる
- 体重・食事・トレーニングの統合グラフを表示する

技術スタック：Next.js、Supabase、Vercel、LINE Messaging API

まずデータベースの設計から始めたい。どんなテーブルが必要か教えて。
```

すると、こんな感じでテーブル設計を提案してくれる：

```sql
-- トレーナー
trainers (id, name, email, created_at)

-- クライアント
clients (id, trainer_id, name, line_user_id, course, goal, created_at)

-- 体重記録
body_records (id, client_id, weight, body_fat, muscle_mass, date)

-- 食事記録
meal_records (id, client_id, image_url, analysis_result, calories, protein, carbs, fat, date)

-- トレーニング記録
training_records (id, client_id, image_url, analysis_result, exercises, date)
```

これを見て「このカラムも欲しい」「これはいらない」と会話しながら修正する。
コードは自分で書かなくていい。Claude Codeが全部書いてくれる。

---

## Step 3: データベースを作る

設計が決まったら、SupabaseのSQL Editorに貼り付けて実行するだけ。

Claude Codeに「Supabase用のSQL作って」と言えば、そのまま使えるコードが出てくる。

**注意点：** Row Level Security（RLS）の設定を忘れずに。
「RLSも設定してほしい。トレーナーは自分のクライアントのデータしか見えないようにして」とClaudeに伝えれば設定コードを書いてくれる。

---

## Step 4: LINEボットを繋げる

LINE Developersでチャンネルを作成する。

1. `https://developers.line.biz` にアクセス
2. 新しいプロバイダーを作成
3. Messaging APIチャンネルを作成
4. チャンネルアクセストークン（長期）をコピー

Claude Codeに渡すもの：
- チャンネルアクセストークン
- チャンネルシークレット

「このトークンを使って、LINEにメッセージが来たら受け取るWebhookを実装して。Next.jsのAPIルートで作って」と伝えれば実装してくれる。

---

## Step 5: AIのスクショ解析を実装する

ここがClient Fitの核心だ。

クライアントがスクショを送ってきたら、Claude APIがそれを解析して構造化データに変換する。

Claude Codeに伝えること：
```
LINEから画像が送られてきたら、Claude APIのVision機能で解析して。
食事の画像なら：カロリー、タンパク質、炭水化物、脂質を抽出。
筋トレアプリのスクショなら：種目名、セット数、重量、レップ数を抽出。
結果をJSONで返して、Supabaseに保存して。
```

1枚の解析コストは約1.1円。1日10枚送っても月330円程度。

---

## Step 6: ダッシュボードを作る

データが溜まり始めたら、それを可視化する画面を作る。

Claude Codeへの指示例：
```
トレーナー向けのダッシュボードを作って。

表示したいもの：
- 全クライアントの今週の記録状況（記録あり/なし）
- 各クライアントの体重推移グラフ（rechartで実装）
- 今日のタンパク質達成率
- 先週比較
```

グラフはrechartというライブラリを使うと綺麗に作れる。
「rechartでこういうグラフ作って」と言うだけでコードを書いてくれる。

---

## Step 7: Vercelで公開する

GitHubにコードをプッシュして、Vercelと連携するだけ。

1. GitHubにリポジトリを作成
2. Vercelの管理画面から「Import Project」
3. 環境変数（SupabaseのURLとキー、LINE APIのキー、AnthropicのAPIキー）を設定
4. デプロイ

5分で完成する。URLが発行されるので、そのURLをLINEのWebhookに設定する。
