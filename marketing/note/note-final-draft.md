# パーソナルトレーナーのクライアント管理がExcelで限界に達したので、AIツールを作った話【作り方も全公開】

---

NSCA-CPTを持っていて、自分でも週5でトレーニングしている。
副業でパーソナルトレーニングも数人見ている。

そんな自分が、ずっと気になっていた課題がある。

---

## クライアント管理が、手作業すぎる。

食事の写真をLINEで送ってもらって、目視でチェックする。
体重の報告も個人LINEで受け取る。
それをExcelに手打ちして、毎週手動でレポートを作って送る。

数人ならなんとかなる。でもこれが10人、20人と増えたとき、管理が破綻するのは目に見えていた。

しかもクライアント側も大変だ。あすけんで食事を記録して、筋トレMemoでトレーニングを記録して、さらにトレーナーへの報告もLINEで別途やる。同じデータを何度も入力するようなものだ。

「タンパク質が足りなかった日の翌日、ベンチプレスの重量はどうだったか」
「体重が3日間動かないとき、原因は食事側なのかトレーニング側なのか」

これを確かめようとすると、あすけんの画面と筋トレMemoの画面を行き来して、頭の中で繋げるしかない。データはちゃんと記録しているのに、答えは結局「勘」になる。

---

## 解決策が、どこにもなかった

同じ悩みを持つトレーナーは多いはずだと思って、解決できるアプリを探した。

Gymz、Trainerize、TrueCoach。あすけんの法人プラン。一通り調べた。

2つの問題があった。

**1つ目は、高すぎること。** トレーナー向けの管理ツールは月額数千円〜数万円かかる。副業で数人を見ているレベルだと、ツール代が利益を食う。

**2つ目は、「なぜ体重が変わらないのか」を教えてくれるものがゼロだったこと。** どのアプリも「記録する」か「見せる」までで、食事×トレーニング×体重の相関分析まではしてくれない。

ないなら、作るしかないと思った。

---

## Claude Codeで作った

プログラミングはほぼやったことがない。

ただClaude Codeというツールを最近知った。AIと会話しながらアプリを作れるツールで、コードが書けなくても「こういうものを作りたい」と日本語で伝えると、AIがコードに変換してくれる。

やってみたら、思っていたより全然できた。

作ったのが **Client Fit** だ。

<!-- 📸 ここにClient Fitダッシュボードのスクショを入れる（トレーナー管理画面） -->

**クライアントはClient Fit公式のLINEにスクショを送るだけ。** あすけんのスクショ、筋トレMemoのスクショ、どちらもAIが自動解析してダッシュボードに反映される。クライアントの操作はこれだけ。手入力ゼロ。

<!-- 📸 ここにLINEスクショ送信→ダッシュボード反映の流れを2枚の静止画で入れる -->

**トレーナー側には翌朝、全クライアントの1日サマリーが届く。** 食事の概要、トレーニング内容、体重の変動。これを見てコメントをワンタップでクライアントのLINEに送れる。

推移グラフで体重×カロリー収支の相関、筋肉量×週間タンパク質平均も自動で見える。まとめレポートもボタン1つでクライアントのLINEに送信できる。

「先週体重が増えたのは脚トレ後のむくみです」
「タンパク質が不足した週は翌週の挙上重量が落ちる傾向があります」

こういうことを、アプリが教えてくれるようになった。

対応しているのはあすけん、筋トレMemo、STRONG、FiNC、カロミルなど主要アプリのスクショ全般だ。クライアントが今使っているアプリのスクショをそのまま送ればいい。乗り換えの必要はない。

---

## Gymzの数分の1の価格

既存のトレーナー向け管理ツールは、機能が充実している分だけ月額も高い。

Client Fitは逆のアプローチを取った。**エントリーは無料。** 追加機能を使っても数百円レベル。副業でクライアントを数人見ているトレーナーでも、ツール代を気にせず導入できる。

「食事管理とトレーニング管理がLINEスクショで一括管理できて、翌朝サマリーが届いて、ボタン1つでレポートを送れる」— これだけで十分な人には、十分すぎるツールだと思う。

---

## 同じものを自分でも作れる

使ったツールと費用はこれだけだ。

| ツール | 用途 | 費用 |
|--------|------|------|
| Claude Code | AIと一緒にコードを書く | 月$20程度 |
| Next.js | Webアプリの土台 | 無料 |
| Supabase | データベース | 無料枠あり |
| Vercel | アプリの公開 | 無料枠あり |
| LINE Developers | LINEボット | 無料 |

合計：月2,000〜3,000円。集中してやれば2〜3日で動くものが作れる。詰まったらClaude Codeに貼り付ければほぼ解決する。

---

### Step 1: 環境を整える（30分）

Node.jsをインストールする。`https://nodejs.org` からLTS版。

Claude Codeをインストール：

```bash
npm install -g @anthropic-ai/claude-code
```

Anthropicのアカウントでログインする。

Supabase（`https://supabase.com`）とVercel（`https://vercel.com`）のアカウントを作成する。

---

### Step 2: Claude Codeに設計を相談する（1時間）

Claude Codeを起動して、こう伝える：

```
パーソナルトレーナー向けのクライアント管理アプリを作りたい。

要件：
- クライアントがLINEにスクショを送るとAIが食事や筋トレ内容を解析する
- トレーナーがダッシュボードで全クライアントの状況を一覧できる
- 翌朝に1日のサマリーがトレーナーに届く
- コメントやレポートをLINE経由でクライアントに送れる

技術スタック：Next.js、Supabase、Vercel、LINE Messaging API

まずデータベースの設計から始めたい。
```

テーブル構成を提案してくれる。「このカラムも欲しい」「これはいらない」と会話しながら修正する。コードは自分で書かなくていい。

---

### Step 3: データベースを作る（1〜2時間）

Claude Codeが出したSQL設計をSupabaseのSQL Editorに貼り付けて実行するだけ。

「RLSも設定してほしい。トレーナーは自分のクライアントのデータしか見えないようにして」と伝えれば、セキュリティ設定も書いてくれる。

---

### Step 4: LINEボットを繋げる（1〜2時間）

LINE Developersでチャンネルを作成する。

1. `https://developers.line.biz` でアカウント作成
2. Messaging APIチャンネルを作成
3. チャンネルアクセストークンをコピー

Claude Codeに「このトークンを使ってLINEのWebhookをNext.jsのAPIルートで実装して」と伝えれば実装してくれる。

---

### Step 5: AIのスクショ解析を実装する（2〜3時間）

Claude Codeへの指示：

```
LINEから画像が届いたら、Claude APIのVision機能で解析して。
食事の画像なら：カロリー・タンパク質・炭水化物・脂質を抽出。
筋トレアプリのスクショなら：種目・セット数・重量・レップ数を抽出。
結果をJSONでSupabaseに保存して。
```

1枚の解析コストは約1.1円（画像サイズで変動）。1日10枚送っても月330円程度。

---

### Step 6: トレーナーダッシュボードを作る（2〜4時間）

Claude Codeへの指示例：

```
トレーナー向けの管理ダッシュボードを作って。
全クライアントの今週の記録状況、体重推移グラフ、タンパク質達成率を表示して。
クライアントごとにコメントを送れるボタンも付けて。
グラフはRechartsで実装して。
```

「こういうグラフが欲しい」と言うだけでコードが出てくる。

---

### Step 7: ローカルで確認してからデプロイ（30分）

まずローカルで動作確認する：

```bash
npm run dev
```

問題なければGitHubにコードをプッシュして、Vercelと連携。環境変数を設定してデプロイボタンを押す。URLが発行される。

自分でカスタマイズし続けたい人以外は、Client Fitをそのまま使った方が早い。

トレーナーとして登録して、クライアントを追加するだけ。クライアントにはClient Fit公式LINEを友達追加してもらって、PINを送ってもらえば連携完了。あとはスクショを送ってもらうだけで、翌朝サマリーが届き始める。
