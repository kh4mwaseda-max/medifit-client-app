# パーソナルトレーナーがあすけんと筋トレMemoの限界に気づいてAIで解決した話【作り方も全公開】

---

NSCA-CPTを持っていて、自分でも週5でトレーニングしている。
副業でパーソナルトレーニングも数人見ている。

そんな自分が、ずっと気になっていた課題がある。

---

## あすけんと筋トレMemoは優秀。でも繋がっていない。

食事管理はあすけんを使っている。1300万人が使っているだけあって、日本語の食品データベースが充実していて使いやすい。

筋トレの記録は筋トレMemo。種目・セット・重量・レップ数を日本語で管理できる。フィジーク競技者にも愛用者が多い。

どちらも単体では本当によくできたアプリだ。

ただ、この2つは繋がっていない。

「タンパク質が足りなかった日の翌日、ベンチプレスの重量はどうだったか」
「体重が3日間動かないとき、原因は食事側なのかトレーニング側なのか」

これを確かめようとすると、あすけんの画面と筋トレMemoの画面を行き来して、頭の中で繋げるしかない。データはちゃんと記録しているのに、答えは結局「勘」になる。

クライアントを見ているときも同じだった。
食事の写真をLINEで送ってもらって、目視でチェックする。
体重の報告も個人LINEで受け取る。
それをExcelに手打ちして、毎週手動でレポートを作って送る。

数人ならなんとかなる。でもこれが10人、20人と増えたとき、管理が破綻するのは目に見えていた。

---

## 解決策が、どこにもなかった

同じ悩みを持つトレーナーは多いはずだと思って、解決できるアプリを探した。

国内のあすけん、FiNC、カロミル。海外のTrainerize、TrueCoach。一通り調べた。

どのアプリも「記録する」か「見せる」かまでで、**「なぜ体重が変わらないのか」を説明してくれるものはゼロだった。**

海外のトレーナー管理ツールは日本語非対応で、クライアントに使わせるには敷居が高すぎる。

ないなら、作るしかないと思った。

---

## Claude Codeで作った

プログラミングはほぼやったことがない。

ただClaude Codeというツールを最近知った。AIと会話しながらアプリを作れるツールで、コードが書けなくても「こういうものを作りたい」と日本語で伝えると、AIがコードに変換してくれる。

やってみたら、思っていたより全然できた。

作ったのが **AYF（All Your Fit）** だ。

<!-- 📸 ここにAYFダッシュボードのスクショを入れる（体重×カロリー収支の相関グラフ画面） -->

クライアントがAYF専用のLINEにスクショを送るだけで、AIが自動解析してダッシュボードに反映される。個人LINEには何も入ってこない。体重×カロリー収支の相関グラフ、筋肉量×週間タンパク質平均が自動で見える。週次レポートも自動生成される。

<!-- 📸 ここにLINEスクショ送信→ダッシュボード反映の流れを2枚の静止画で入れる -->

対応しているのはあすけん、筋トレMemo、STRONG、FiNC、カロミルなど主要アプリのスクショ全般だ。特定のアプリに乗り換える必要はない。今使っているアプリのスクショをそのまま送ればいい。

「先週体重が増えたのは脚トレ後のむくみです」
「タンパク質が不足した週は翌週の挙上重量が落ちる傾向があります」

こういうことを、アプリが言ってくれるようになった。

自分で使いながら、「これ確かめたかったやつだ」と毎回思う。

## 同じものを自分でも作れる

使ったツールと費用はこれだけだ。

| ツール | 用途 | 費用 |
|--------|------|------|
| Claude Code | AIと一緒にコードを書く | 月$20程度 |
| Next.js | Webアプリの土台 | 無料 |
| Supabase | データベース | 無料枠あり |
| Vercel | アプリの公開 | 無料枠あり |
| LINE Developers | LINEボット | 無料 |

合計：月2,000〜3,000円。集中してやれば2〜3日で動くものが作れる。詰まったらClaude Codeに貼り付ければほぼ解決する。

---

### Step 1: 環境を整える（30分）

Node.jsをインストールする。`https://nodejs.org` からLTS版。

Claude Codeをインストール：

```bash
npm install -g @anthropic-ai/claude-code
```

Anthropicのアカウントでログインする。

Supabase（`https://supabase.com`）とVercel（`https://vercel.com`）のアカウントを作成する。

---

### Step 2: Claude Codeに設計を相談する（1時間）

Claude Codeを起動して、こう伝える：

```
パーソナルトレーナー向けのクライアント管理アプリを作りたい。

要件：
- クライアントがLINEにスクショを送るとAIが食事や筋トレ内容を解析する
- トレーナーがダッシュボードで全クライアントの状況を一覧できる
- 体重・食事・トレーニングの統合グラフを表示する

技術スタック：Next.js、Supabase、Vercel、LINE Messaging API

まずデータベースの設計から始めたい。
```

テーブル構成を提案してくれる。「このカラムも欲しい」「これはいらない」と会話しながら修正する。コードは自分で書かなくていい。

---

### Step 3: データベースを作る（1〜2時間）

Claude Codeが出したSQL設計をSupabaseのSQL Editorに貼り付けて実行するだけ。

「RLSも設定してほしい。トレーナーは自分のクライアントのデータしか見えないようにして」と伝えれば、セキュリティ設定も書いてくれる。

---

### Step 4: LINEボットを繋げる（1〜2時間）

LINE Developersでチャンネルを作成する。

1. `https://developers.line.biz` でアカウント作成
2. Messaging APIチャンネルを作成
3. チャンネルアクセストークンをコピー

Claude Codeに「このトークンを使ってLINEのWebhookをNext.jsのAPIルートで実装して」と伝えれば実装してくれる。

---

### Step 5: AIのスクショ解析を実装する（2〜3時間）

Claude Codeへの指示：

```
LINEから画像が届いたら、Claude APIのVision機能で解析して。
食事の画像なら：カロリー・タンパク質・炭水化物・脂質を抽出。
筋トレアプリのスクショなら：種目・セット数・重量・レップ数を抽出。
結果をJSONでSupabaseに保存して。
```

1枚の解析コストは約1.1円（画像サイズで変動）。1日10枚送っても月330円程度。

---

### Step 6: ダッシュボードを作る（2〜4時間）

Claude Codeへの指示例：

```
トレーナー向けのダッシュボードを作って。
全クライアントの今週の記録状況、体重推移グラフ、タンパク質達成率を表示して。
グラフはRechartsで実装して。
```

「こういうグラフが欲しい」と言うだけでコードが出てくる。

---

### Step 7: Vercelで公開する（30分）

GitHubにコードをプッシュして、Vercelと連携するだけ。
環境変数（SupabaseのURL・キー、LINE APIのキー、AnthropicのAPIキー）を設定してデプロイボタンを押す。URLが発行される。

手順を読んで「難しそう」と感じた人は、たぶん最後まで辿り着けない。途中で詰まって時間だけ消える体験になりやすい。

自分でカスタマイズし続けたい人以外は、AYFをそのまま使った方が早い。
