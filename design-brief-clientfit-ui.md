# Claude Design 依頼: Client Fit UI/UX 一式

> 作成: 2026-04-20 / 田中美咲（秘書）・佐藤次郎CEO共同
> 用途: Claude Design にコピペして LP + 認証 + ダッシュボード等すべてのUI/UXをまとめて依頼
> 納品後: Claude Code 側で Next.js 16 + React 19 + Tailwind 4 に実装

---

## 依頼内容

**Client Fit の全ウェブUI（LP / 認証 / ダッシュボード / クライアント管理 / クライアント側画面）のデザインを一式でお願いします。**

単発のPSDではなく、一貫したデザインシステムとして組んでください。実装しやすいようコンポーネント粒度で分割納品希望。

---

## 1. サービスの前提

### 一言定義
**「クライアントはスクショを送るだけ。トレーナーは翌朝サマリーを見るだけ。」**

パーソナルトレーナー向けクライアント管理 SaaS。クライアントが あすけん/筋トレMemo/STRONG/タニタ等のスクショを LINE に送るだけで、食事・トレーニング・体組成を自動解析して記録。トレーナーは翌朝届くサマリーを見てコメント・レポートをワンタップでLINE送信できる。

### ターゲット
- **プライマリ**: フリーランスのパーソナルトレーナー（クライアント3〜20名規模）
- **セカンダリ**: 小規模ジムのオーナー

### 料金
- エントリー: 無料
- 追加機能（今後）: 数百円/月

## 2. 2種類のユーザーとフロー

### トレーナー（管理者側）

```
[LP] → [登録] → [ログイン] → [初回セットアップ]
    → [ダッシュボード(クライアント一覧)]
        → [新規クライアント追加]
        → [クライアント詳細]
            → [レポート画面]
        → [設定]
```

### クライアント（被管理側）

```
[招待リンク] → [PIN発行] → [LINE連携] → [クライアントダッシュボード]
                                            ├ 今日タブ
                                            ├ 記録タブ（推移グラフ）
                                            ├ フォトタブ
                                            └ AIタブ
```

## 3. デザインシステム（最初に決めてほしい）

### カラーパレット

| 用途 | 色 | HEX |
|------|-----|-----|
| プライマリ | 青 | `#3b82f6` |
| アクセント | シアン | `#0ea5e9` |
| グラデーション | プライマリ→アクセント | `linear-gradient(135deg, #3b82f6, #0ea5e9)` |
| 背景ダーク（LP用） | 紺 | `#0a0f1e` |
| 背景ライト（ダッシュボード用） | 白寄り | `#f8fafc` |
| テキスト白 | 見出し | `#ffffff` |
| テキスト濃 | ダッシュボード見出し | `#1e293b` |
| テキスト薄 | 本文 | `#94a3b8` |
| テキスト極薄 | 補足 | `#64748b` |
| 成功 | 緑 | `#10b981` |
| 警告 | 琥珀 | `#f59e0b` |
| 危険 | 赤 | `#ef4444` |

### タイポグラフィ

- フォント: System UI（日本語対応前提）
- 見出し: font-black（900）
- 本文: font-normal（400）
- 数値強調: font-mono（推移グラフ等）

### デザイン方針

- **LP**: ダークテーマ（`#0a0f1e`）でプロフェッショナル・データドリブン感
- **認証/ダッシュボード**: ライトテーマ（`#f8fafc`）でクリーン・業務ツール感
- **参考**: Linear / Vercel / Notion / Linear Design
- **避ける**: 筋肉アピール / 派手なオレンジ / チープな装飾

### コンポーネント原則

- アイコン: Lucide React
- ボタン: グラデーションCTA（プライマリ→アクセント）+ セカンダリ（outline）
- カード: 白BG + `shadow-sm` + `rounded-2xl`
- フォーム: 入力欄は `border-slate-200` / focus で `ring-blue-500`
- モバイルファースト（Tailwind ブレークポイント準拠）

## 4. 画面一覧と個別要件（全12画面）

### ① `/` — LP（ランディングページ）

**目的**: トレーナーに登録を促す

**背景**: ダークテーマ（`#0a0f1e`）

**構成セクション**:
1. **Hero** — キャッチ「クライアント管理を、もっと簡単に。」/ サブコピー / CTA「無料で始める」/ ダッシュボードモック（デバイスフレーム付き）
2. **Pain Points** — 3〜4課題（アイコン付きカード）
   - 食事チェックが手作業
   - データが LINE・Excel・メモに分散
   - レポート作成が毎週手作業
   - 既存SaaSが高い（月数千〜数万円）
3. **How It Works** — 3ステップ
   - ① トレーナー登録（1分）
   - ② クライアント招待（リンク発行→スマホでPIN入力）
   - ③ スクショ受信で自動記録（LINE経由）
4. **Features** — 9機能のカード
   - LINE スクショ自動解析
   - クライアントダッシュボード
   - デイリーサマリー配信
   - ワンタップLINEコメント
   - 招待リンクで簡単追加
   - LINEオンボーディング自動化
   - AIアセスメント
   - 週次/月次レポート
   - 公式LINE経由（個人連絡先不要）
5. **Screenshots** — ダッシュボード / デイリーサマリー / クライアント画面のデバイスフレーム付きモック
6. **Pricing** — 無料プランを大きく訴求
7. **Comparison** — Gymz等との比較表（料金・導入の簡単さ・LINE連携）
8. **FAQ** — 5問
9. **Final CTA** — 再登録ボタン + 「個人LINE交換不要」訴求
10. **Footer** — 運営・法的情報・問い合わせ

---

### ② `/trainer/register` — トレーナー登録

**目的**: 新規トレーナーが登録する

**背景**: ライトテーマ

**要素**:
- Client Fit ロゴ（ページ上部中央）
- カード型フォーム
  - 名前
  - メールアドレス
  - パスワード
- 「登録して始める」CTAボタン（グラデーション）
- 「すでにアカウントをお持ちの方はログイン」リンク
- 利用規約・プライバシーポリシーへの同意チェック

**状態**:
- 入力エラー時のインライン表示
- ローディング中のボタンスピナー
- 成功時はダッシュボードへリダイレクト

---

### ③ `/trainer/login` — トレーナーログイン

**目的**: 既存トレーナーがログイン

**要素**:
- ロゴ
- メール + パスワード入力
- ログインボタン（グラデーション）
- 「マジックリンクでログイン」リンク（メールでワンタイムURL送信）
- 「アカウントをお持ちでない方は登録」リンク
- 5回失敗で15分ロックの警告表示

---

### ④ `/trainer/setup` — 初回セットアップ（オンボーディング）

**目的**: 登録直後のトレーナーがLINE連携・プロフィール設定を行う

**要素**:
- ステップ進捗インジケーター（3ステップ）
- Step 1: プロフィール（肩書・プロフィール写真・自己紹介）
- Step 2: LINE公式アカウント連携（QRコード提示 + 連携確認）
- Step 3: 最初のクライアントを招待（任意・スキップ可）
- 完了画面 → ダッシュボードへ

---

### ⑤ `/trainer` — トレーナーダッシュボード（クライアント一覧）

**目的**: 全クライアントを一元管理

**要素**:
- **サイドバー**（PC） / ハンバーガー（SP）: ダッシュボード / 設定 / ログアウト
- **ヘッダー**: Client Fit ロゴ / ユーザーアイコン
- **メインエリア**:
  - サマリーカード4つ（総クライアント数 / 今日の記録件数 / 要対応フラグ件数 / LINE月使用枚数）
  - 「クライアント追加」CTA（右上）
  - **クライアント一覧テーブル**:
    - カラム: 名前 / 最終活動日 / 体重推移（ミニグラフ） / 要対応フラグ / アクション
    - 行クリックで詳細画面へ
    - ソート: 最終活動日降順デフォルト
    - 要対応フラグは色（赤/黄）で視認化

---

### ⑥ `/trainer/clients/new` — 新規クライアント追加

**目的**: クライアントを登録して招待リンクorPINを発行

**要素**:
- カード型フォーム
  - 名前
  - 目標（コース選択: 減量 / ボディメイク / バルクアップ / 健康維持 / 大会準備）
  - 開始日
  - 目標体重・目標体脂肪率（任意）
- 「登録して招待リンクを発行」CTA
- 発行後の画面:
  - 招待リンク（`/join/[token]`）のコピーボタン
  - QRコード表示
  - 「LINEで送る」ボタン
  - PIN表示（バックアップ）

---

### ⑦ `/trainer/clients/[id]` — クライアント詳細

**目的**: 個別クライアントのデータ閲覧 + 記録入力 + アセスメント生成

**レイアウト**: 2カラム（PC）、タブ切替（SP）

**左カラム（クライアント情報）**:
- プロフィールカード（写真・名前・目標・開始日・経過日数）
- 目標進捗バー
- クイックアクション（LINEコメント送信 / レポート生成 / 編集）

**右カラム（データタブ）**:
- **今日タブ**: 当日の体重・食事カロリー・トレーニング概要 + トレーナーメモ入力
- **推移タブ**: 体重/体脂肪/筋肉量のグラフ（7/30/90日）、トレーニングボリューム、食事PFC
- **フォトタブ**: Before/After写真 + サイドバイサイド比較スライダー
- **AIタブ**: アセスメントレポート / 疾患リスクインジケーター / 改善策 / レコメンデーション（サプリ・食品）
- **記録入力タブ**: 身体計測手動入力 / トレーニングセッション手動入力

---

### ⑧ `/trainer/clients/[id]/report` — レポート生成・送信

**目的**: 週次/月次レポートを自動生成してLINE送信

**要素**:
- 期間切替（週次 / 月次 / カスタム）
- 自動生成レポートのプレビュー
  - サマリー文章（AI生成）
  - 推移グラフ
  - トレーニング記録サマリー
  - 食事記録サマリー
  - トレーナーコメント欄（編集可）
- 「LINEで送信」CTA（送信確認モーダル付き）
- PDF出力ボタン

---

### ⑨ `/trainer/settings` — トレーナー設定

**目的**: プロフィール・LINE連携・レート制限等の管理

**要素（タブ切替）**:
- プロフィール: 名前・肩書・写真・自己紹介
- LINE連携: チャネル情報・月使用枚数（プログレスバー）・警告閾値
- セキュリティ: パスワード変更・セッション一覧
- 通知: デイリーサマリー配信時刻
- 料金・請求: 無料プラン表示 + 追加機能（将来）
- 危険ゾーン: アカウント削除

---

### ⑩ `/client/[id]` — クライアントダッシュボード

**目的**: クライアント本人が自身のデータを見る

**背景**: ライトテーマ（親しみやすさ重視）

**要素**:
- シンプルヘッダー（名前 + ログアウト）
- 進捗カード（目標まであと◯kg / ◯日）
- タブ切替:
  - **今日**: 当日の食事・トレーニング・体重カード
  - **記録**: 推移グラフ（期間切替）
  - **フォト**: Before/After タイムライン
  - **AI**: アセスメントレポート（読みやすい形式で）
- フッターCTA: 「LINEにスクショを送る」使い方ガイド

---

### ⑪ `/join/[token]` — 招待リンク受諾

**目的**: トレーナーから送られたリンクをスマホで開いて PIN 取得

**レイアウト**: モバイル優先・シンプル・1画面完結

**要素**:
- トレーナー名・トレーナーアイコン
- 「◯◯さんがあなたを招待しています」メッセージ
- 名前入力（任意・事前入力されていれば確認）
- 「受諾して PIN を発行」CTA
- PIN発行後:
  - 6桁PINを大きく表示（コピーボタン）
  - LINE公式アカウント追加QRコード
  - 「LINEでこのPINを送信してください」案内
  - 使い方3ステップ図解

---

### ⑫ `/demo` — デモダッシュボード

**目的**: 未登録ユーザーがサンプルデータで試せる

**要素**:
- `/client/[id]` と同等のUI
- サンプルデータで動作
- トップに「これはデモ画面です」バナー + 登録CTA

## 5. 共通コンポーネント（デザインシステムに含めてほしい）

### ナビゲーション
- **トレーナーサイドバー**: アイコン + ラベル（ホーム / クライアント / レポート / 設定 / ログアウト）
- **モバイルボトムナビ**: 4タブ（ホーム / クライアント / 通知 / プロフィール）

### ボタン
- `<Button variant="primary">` — グラデーション（CTA用）
- `<Button variant="secondary">` — outline
- `<Button variant="ghost">` — 背景なし
- `<Button variant="danger">` — 赤
- サイズ: sm / md / lg
- ローディング状態（スピナー）・無効状態

### フォーム
- `<Input>`, `<Select>`, `<Textarea>`, `<Checkbox>`, `<RadioGroup>`
- エラーメッセージ表示
- ラベル + 補足テキスト

### カード
- `<Card>` — 白BG + shadow-sm + rounded-2xl
- `<StatCard>` — 数値 + ラベル + 前日比矢印
- `<ClientCard>` — クライアント情報1件分

### データ表示
- `<LineChart>` — 推移グラフ（Recharts）
- `<BarChart>` — ボリュームグラフ
- `<Table>` — ソート・ページネーション対応
- `<RiskIndicator>` — 疾患リスクの4段階表示

### フィードバック
- `<Toast>` — 成功・エラー・情報通知
- `<Modal>` — 確認ダイアログ
- `<EmptyState>` — データなし時のプレースホルダー
- `<LoadingSkeleton>` — ローディング

### その他
- `<Logo>` — ロゴマーク（dark/light テーマ対応）
- `<Avatar>` — イニシャルアイコン・画像アップロード
- `<Badge>` — ステータス表示
