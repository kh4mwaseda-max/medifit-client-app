# Client Fit（CF）評価・実装ルーティン ハーネスプロンプト
> このファイルをClaude Codeに渡すと、現状評価→優先タスク特定→実装→ステータス更新まで自走します。

---

あなたはClient Fit（CF）のプロダクト開発担当として、以下の①〜⑤のサイクルを1周完遂してください。途中で止まらず実行してください。

**基準ドキュメント**:
- 仕様・タスク管理: `../../status.md`
- コード: `src/` 以下

---

## ① 現状評価（Evaluate）

以下の順で現状を把握してください。

**1-A. ステータス確認**
`../../status.md` を読み、「残実装タスク」と「次アクション（Claude Code側）」のうち未完了（⬜）のものを列挙する。

**1-B. コード照合**
各タスクに対応するファイルが存在するか確認する。

```
確認対象（例）:
- recommendation-engine.ts → src/lib/recommendation-engine.ts
- health-data-connector.ts → src/lib/health-data-connector.ts
- /register ページ → src/app/(register)/page.tsx 等
```

**1-C. 評価レポートを出力する**

```
## 現状評価レポート

### 実装済みと判定したもの
- [ファイルパス] → 〇〇タスクに対応

### 未実装 / 不完全と判定したもの
- [タスク名]: 理由（ファイルなし / スタブのみ / ロジック未実装 等）

### 今サイクルで実装すべき優先タスク（1〜2個）
1. [タスク名]: 理由
```

---

## ② 優先タスク選定（Prioritize）

以下の基準で今サイクルの実装対象を1〜2個選んでください。

**優先順位の基準（高い順）:**
1. ユーザーが触れる機能（ダッシュボード・オンボーディング・LINE連携）
2. 他タスクのブロッカーになっている処理
3. 稲川オーナーへのデモに必要なもの
4. コード量が少なく完了できるもの

**選定の除外条件:**
- 稲川さん側の作業待ち（Supabase・LINE設定・申請など）
- 外部APIキーが必要で未設定のもの（マイナポータル等）

---

## ③ 実装（Implement）

選定したタスクを実装してください。

**実装の原則:**
- 既存コードのスタイル・命名規則を必ず踏襲する
- `src/lib/` のユーティリティを積極的に再利用する
- TypeScript型は既存の `database.types.ts` を参照する
- `TODO:` コメントを残さない。未実装のブランチは書かない
- Claude API を使う場合は `claude-sonnet-4-6` モデルを使用する
- エラーハンドリングはユーザーに見えるメッセージとserver-side logの両方を書く

**実装後の確認:**
```bash
# TypeScriptコンパイルエラーがないか確認
npx tsc --noEmit

# lintエラーがないか確認
npx eslint src/ --ext .ts,.tsx
```

エラーがあれば修正してから次のステップへ進む。
